# Security Policy

## Reporting Protocol Vulnerabilities

If you discover a security vulnerability in the AIRGP specification itself (e.g., a flaw in the wire protocol design, audit ledger integrity, or identity/attestation mechanism), please report it responsibly.

## How to Report

Email: security@thinkneo.ai

Include:
- Description of the vulnerability
- Affected specification section(s)
- Potential impact assessment
- Suggested mitigation (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 business days
- **Resolution plan:** Within 30 business days
- **Public disclosure:** Coordinated with reporter, typically 90 days after report

## Scope

This policy covers vulnerabilities in:
- The AIRGP specification design
- JSON Schema definitions
- Reference architecture patterns
- Conformance test procedures

For vulnerabilities in specific implementations of AIRGP, contact the implementation vendor directly.
