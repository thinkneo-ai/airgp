# Contributing to AIRGP

Thank you for your interest in contributing to the Artificial Intelligence Runtime Governance Protocol.

## How to Contribute

### Specification Feedback

- Open a GitHub Issue with the `feedback` label
- Describe the section, the concern, and a proposed resolution
- The working group reviews all feedback during quarterly cycles

### Schema Contributions

- Fork this repository
- Add or modify schemas in `schemas/`
- Ensure all schemas validate against JSON Schema Draft 2020-12
- Submit a Pull Request with a clear description

### Example Contributions

- Add examples to `examples/` that demonstrate real-world AIRGP usage
- Each example must validate against the corresponding schema
- Include a brief comment header explaining the scenario

### Editorial Corrections

- Typos, formatting, and clarity improvements are welcome as Pull Requests
- Use the `editorial` label

## What Cannot Be Changed via PR

The specification text in `SPEC.md` and `spec/v1.0/` is governed by the Steering Committee amendment process (see `GOVERNANCE.md`). Pull Requests that modify normative specification text will be redirected to the formal amendment process.
